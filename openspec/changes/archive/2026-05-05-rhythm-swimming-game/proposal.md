## 为什么

当前市场上音乐节奏类游戏同质化严重，缺少与体育运动真实结合的玩法创新。将音乐节奏判定与游泳运动的泳频（划水频率）和速度相结合，可以创造一种全新的竞技体验：玩家需要在音乐节拍上精准"划水"，节奏越准、泳速越快，从而在10人房间中争夺名次。这种设计既保留了音游的核心乐趣，又引入了体育竞速的紧张感和策略性。

## 变更内容

- **新增** 10人实时竞技房间系统，支持匹配、等待、倒计时开赛
- **新增** 游泳赛道可视化系统，每位玩家在独立泳道中前进
- **新增** 音乐节奏判定引擎，将玩家输入与音乐节拍进行匹配评分（Perfect / Great / Good / Miss）
- **新增** 泳频-速度映射模型：玩家的节奏准确度决定划水频率，连续准确输入产生速度加成
- **新增** 体力与耐力系统：过快/过慢的节奏会消耗额外体力，影响后半程表现
- **新增** 冲刺机制：特定音乐高潮段落触发冲刺阶段，节奏判定权重提升
- **新增** 结算与排名系统：基于到达终点的时间进行排名，显示节奏准确率统计

## 功能 (Capabilities)

### 新增功能

- `room-system`: 10人房间匹配、状态管理、倒计时与同步开赛机制
- `swimming-track`: 游泳赛道可视化，包含泳道分配、玩家位置同步和动画表现
- `rhythm-engine`: 音乐节奏判定核心引擎，处理音符生成、输入检测和评分逻辑
- `swim-speed-model`: 泳频-速度映射模型，将节奏准确度转化为游泳速度和加速度
- `stamina-system`: 体力与耐力系统，管理体力消耗、恢复和对速度的影响
- `sprint-mechanic`: 冲刺机制，在音乐高潮段落触发特殊高倍判定阶段
- `result-ranking`: 比赛结算与排名系统，计算最终成绩并展示统计数据

## 影响

- **前端**：需要新增游泳赛道 UI、节奏输入面板、玩家状态 HUD、结算界面
- **后端**：需要新增房间服务、比赛状态同步服务、排名计算服务
- **网络**：需要设计低延迟的输入同步协议（节奏游戏对延迟敏感）
- **音频**：需要音乐节拍数据预处理管线，支持BPM检测和节拍标注
- **资源**：需要游泳动画资源、音乐曲库、UI 美术资源
- **依赖**：可能需要引入音频分析库用于BPM检测和节拍提取
