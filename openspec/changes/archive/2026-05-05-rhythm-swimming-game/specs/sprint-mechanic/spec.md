## 新增需求

### 需求:冲刺段落定义
系统必须支持在曲目节拍数据中预标注冲刺段落。每首曲目必须有1-3个冲刺段落，每个段落持续时间为8-16秒。冲刺段落必须与音乐高潮部分对应。

#### 场景:冲刺段落触发
- **当** 比赛时间轴进入预标注的冲刺段落起始时间
- **那么** 系统必须向所有玩家同时广播冲刺开始，所有客户端必须同步进入冲刺模式

#### 场景:冲刺段落结束
- **当** 比赛时间轴到达冲刺段落的结束时间
- **那么** 系统必须结束冲刺模式，所有计算恢复为正常模式

### 需求:冲刺判定权重
在冲刺段落期间，所有判定对泳速的影响必须翻倍。具体规则：冲刺期间的划水效率系数 = 正常效率 × 2.0（上限为2.0），Miss惩罚同样翻倍。

#### 场景:冲刺期间Perfect
- **当** 玩家在冲刺段落中获得Perfect判定
- **那么** 该拍的划水效率必须为2.0（1.0 × 2.0），产生双倍推进距离

#### 场景:冲刺期间Miss
- **当** 玩家在冲刺段落中获得Miss判定
- **那么** 该拍效率仍为0.0，且体力额外消耗必须翻倍（从3点变为6点）

### 需求:冲刺视觉与音效表现
进入冲刺段落时，系统必须提供显著的视觉和音效变化，让玩家明确感知到冲刺状态。

#### 场景:冲刺开始视觉效果
- **当** 冲刺段落开始
- **那么** 系统必须播放冲刺开始特效（如水花加大、泳道发光），并在界面显示"SPRINT!"提示文字

#### 场景:冲刺期间持续表现
- **当** 冲刺段落进行中
- **那么** 界面必须持续显示冲刺状态指示器（如边框发光/颜色变化），音符的视觉样式必须区别于正常段落

#### 场景:冲刺即将开始预警
- **当** 距离冲刺段落开始还有3秒
- **那么** 系统必须显示预警提示（如"3...2...1...SPRINT!"倒计时），让玩家做好准备

### 需求:冲刺段落公平性保障
所有玩家必须同时进入和退出冲刺段落。冲刺期间的加成规则对所有玩家完全一致，禁止任何差异化处理。

#### 场景:全员同步进入冲刺
- **当** 冲刺段落触发
- **那么** 所有存活（未放弃）的玩家必须在同一逻辑帧进入冲刺状态

#### 场景:断线玩家冲刺处理
- **当** 某玩家在冲刺段落期间处于断线状态
- **那么** 该玩家不获得任何冲刺收益，其位置保持静止
