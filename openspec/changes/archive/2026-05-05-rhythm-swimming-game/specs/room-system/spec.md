## 新增需求

### 需求:房间匹配与创建
系统必须支持10人房间的自动匹配，玩家点击匹配后系统必须将其加入一个等待中的房间或创建新房间。每个房间的容量必须为10人，禁止超过10人加入同一房间。

#### 场景:玩家发起匹配
- **当** 玩家点击"开始匹配"按钮
- **那么** 系统将玩家加入一个未满员的等待房间，若无可用房间则创建新房间

#### 场景:房间满员
- **当** 第10名玩家加入房间
- **那么** 系统立即关闭房间匹配入口，开始倒计时流程

#### 场景:匹配超时
- **当** 房间在60秒内未达到10人
- **那么** 系统必须以当前人数（最少3人）开赛；若不足3人则解散房间并通知玩家重新匹配

### 需求:倒计时与同步开赛
房间满员或匹配超时后，系统必须执行统一倒计时，所有玩家必须在同一时刻开始比赛。倒计时期间禁止玩家离开房间。

#### 场景:正常倒计时开赛
- **当** 房间触发开赛条件
- **那么** 系统向所有玩家广播3秒倒计时，倒计时结束后所有客户端同步开始比赛

#### 场景:倒计时期间玩家断线
- **当** 某玩家在倒计时期间断线
- **那么** 系统必须继续倒计时并正常开赛，断线玩家位置保留，其角色以静止状态参赛

### 需求:房间状态管理
系统必须维护房间完整生命周期状态：等待中 → 倒计时 → 比赛中 → 结算中 → 已关闭。状态转换必须严格按顺序执行，禁止跳跃状态。

#### 场景:比赛正常结束
- **当** 所有玩家到达终点或比赛时间耗尽
- **那么** 房间状态变为"结算中"，计算排名后广播结果

#### 场景:所有玩家断线
- **当** 比赛中所有玩家断线
- **那么** 系统必须在30秒等待后关闭房间并释放资源

### 需求:断线重连
系统必须支持比赛期间玩家断线重连。重连后必须恢复玩家当前比赛进度。

#### 场景:玩家断线后重连
- **当** 玩家在比赛中断线并在30秒内重连
- **那么** 系统必须将当前比赛状态快照发送给该玩家，恢复其参赛状态

#### 场景:玩家超时未重连
- **当** 玩家断线超过30秒
- **那么** 系统必须标记该玩家为"已放弃"，其角色停留在断线位置不再前进
