## 新增需求

### 需求:音符生成与时间轴
系统必须根据预处理的曲目节拍数据，在比赛开始后按时间轴生成音符。音符必须提前一定时间出现在屏幕上，给予玩家视觉预判时间。

#### 场景:音符按节拍生成
- **当** 比赛开始且音乐播放
- **那么** 系统必须按照曲目节拍数据在对应时间点生成音符，音符提前出现时间（前置时间）必须根据曲目BPM动态计算（范围：0.8秒-1.5秒）

#### 场景:不同类型音符
- **当** 曲目数据中标注了不同类型的节拍（单拍、长拍、双拍）
- **那么** 系统必须生成对应类型的音符控件，每种类型有独立的视觉表现和输入方式

### 需求:输入检测与时间戳记录
系统必须精确检测玩家的输入操作，并记录输入时间戳（精度必须达到毫秒级）。输入类型必须包括：单次点击、长按、双击。

#### 场景:单次点击输入
- **当** 玩家执行单次点击操作
- **那么** 系统必须记录点击时间戳，并在1帧内将其传递给判定模块

#### 场景:长按输入
- **当** 玩家按住输入并在指定时间后松开
- **那么** 系统必须记录按下时间戳和松开时间戳，长按持续时间必须与曲目中长拍的标注时长进行比对

#### 场景:输入防抖
- **当** 玩家在50ms内连续触发多次输入
- **那么** 系统必须仅保留第一次有效输入，后续输入必须被忽略

### 需求:节奏判定评分
系统必须将玩家输入时间与目标节拍时间进行比对，根据时间差产生判定等级。判定等级必须分为：Perfect（±30ms）、Great（±60ms）、Good（±100ms）、Miss（>100ms或无输入）。

#### 场景:Perfect判定
- **当** 玩家输入时间与节拍时间差在±30ms以内
- **那么** 系统必须判定为Perfect，显示对应的视觉特效和音效反馈

#### 场景:Miss判定（无输入）
- **当** 节拍时间已过且超过100ms仍无玩家输入
- **那么** 系统必须判定为Miss，音符消失并显示Miss反馈

#### 场景:判定结果立即反馈
- **当** 判定完成
- **那么** 系统必须在判定完成后16ms内（1帧）向玩家展示判定等级文字和对应视觉特效

### 需求:连击系统
系统必须追踪玩家连续获得非Miss判定的次数（连击数）。连击数必须在获得Miss判定时重置为0。

#### 场景:连击累加
- **当** 玩家连续获得Perfect或Great或Good判定
- **那么** 连击计数器必须加1，并在界面上实时显示当前连击数

#### 场景:连击中断
- **当** 玩家获得Miss判定
- **那么** 连击计数器必须立即重置为0，并播放连击中断的视觉/音效反馈

#### 场景:高连击里程碑提示
- **当** 连击数达到特定里程碑（10、25、50、100）
- **那么** 系统必须播放特殊的里程碑庆祝特效

### 需求:音画同步保障
系统必须确保音乐播放与音符显示之间的同步误差不超过±5ms。如果检测到偏移，系统必须自动校正。

#### 场景:音频延迟校正
- **当** 系统检测到音频播放与预定时间轴存在偏移
- **那么** 系统必须调整音符显示时间或音频播放位置，确保误差回到±5ms以内

#### 场景:低帧率场景处理
- **当** 客户端帧率降至30fps以下
- **那么** 系统必须优先保障判定时间轴的准确性，可降低视觉特效但禁止影响判定精度
