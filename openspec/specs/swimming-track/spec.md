## 新增需求

### 需求:泳道分配与显示
系统必须为每位参赛玩家分配独立泳道。泳道必须按从上到下（或从左到右）的顺序排列，本地玩家的泳道必须始终高亮显示。

#### 场景:比赛开始分配泳道
- **当** 比赛倒计时结束、正式开赛
- **那么** 系统必须为每位玩家分配唯一编号的泳道（1-10），并在所有客户端上渲染完整的赛道视图

#### 场景:本地玩家泳道高亮
- **当** 比赛界面渲染完成
- **那么** 本地玩家所在泳道必须以区别于其他泳道的视觉样式（高亮边框/不同底色）展示

### 需求:玩家位置同步与显示
系统必须实时同步所有玩家在赛道上的位置，位置以"已游距离/总距离"百分比表示。所有客户端必须能看到所有玩家的相对位置。

#### 场景:位置实时更新
- **当** 任意玩家的泳速发生变化
- **那么** 系统必须在100ms内将该玩家的新位置同步到所有客户端

#### 场景:玩家到达终点
- **当** 玩家的已游距离达到总距离100%
- **那么** 系统必须在该泳道显示"到达"动画，并记录到达时间用于排名

### 需求:游泳动画表现
系统必须根据玩家当前泳速和判定结果播放对应的游泳动画。动画状态必须包括：待命、正常划水、快速划水、冲刺划水、疲劳划水、静止。

#### 场景:根据速度切换动画
- **当** 玩家泳速从正常区间进入快速区间
- **那么** 角色动画必须在0.2秒内平滑过渡到快速划水动画

#### 场景:Miss判定动画反馈
- **当** 玩家获得Miss判定
- **那么** 角色必须播放一个明显的"失误"动画表现（如划水中断/溅水），持续不超过0.5秒

### 需求:赛道进度条
系统必须在界面顶部或侧边显示一个缩略进度条，展示所有玩家的相对位置和名次。

#### 场景:进度条实时刷新
- **当** 比赛进行中
- **那么** 进度条必须以不低于10fps的频率刷新所有玩家的位置标记

#### 场景:显示当前名次
- **当** 玩家位置发生变化导致名次变动
- **那么** 进度条上必须更新名次标记，并对本地玩家的名次变动给出视觉/音效提示
