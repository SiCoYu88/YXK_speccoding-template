## 新增需求

### 需求:判定等级到划水效率映射
系统必须将节奏判定等级转换为划水效率系数。映射关系必须为：Perfect=1.0、Great=0.8、Good=0.5、Miss=0.0。

#### 场景:Perfect判定产生最高效率
- **当** 玩家获得Perfect判定
- **那么** 该次划水的效率系数必须为1.0，角色以最大速率前进

#### 场景:Miss判定无推进
- **当** 玩家获得Miss判定
- **那么** 该次划水的效率系数必须为0.0，角色在该拍期间不产生任何前进距离

### 需求:泳速计算公式
系统必须按以下公式计算每拍的实际泳速：`实际泳速 = 基础速率 × 划水效率 × 连击加成系数 × 体力系数`。所有系数必须为非负数。

#### 场景:正常速度计算
- **当** 玩家获得一次判定且无特殊状态
- **那么** 系统必须使用完整公式计算泳速，并将计算结果应用于玩家位置推进

#### 场景:多系数叠加
- **当** 玩家处于高连击且体力充沛状态下获得Perfect判定
- **那么** 所有系数必须正确相乘，最终泳速必须大于任何单一系数的效果

### 需求:连击加成系数
系统必须根据当前连击数提供速度加成。加成规则必须为：0-9连击=1.0x、10-24连击=1.1x、25-49连击=1.2x、50-99连击=1.3x、100+连击=1.5x。

#### 场景:连击达到加成阈值
- **当** 玩家连击数从9增加到10
- **那么** 连击加成系数必须从1.0提升为1.1，立即生效于下一拍的速度计算

#### 场景:连击中断重置加成
- **当** 玩家获得Miss导致连击中断
- **那么** 连击加成系数必须立即重置为1.0

### 需求:速度平滑处理
系统必须对计算得到的泳速进行平滑处理，避免速度在相邻拍之间产生过大的跳变。平滑窗口必须为最近3拍的加权平均。

#### 场景:速度平滑生效
- **当** 玩家从连续Perfect突然获得一次Good
- **那么** 显示的泳速必须平滑过渡而非瞬间骤降，过渡时间不超过0.3秒

#### 场景:Miss后速度变化
- **当** 玩家获得Miss判定
- **那么** 泳速必须在0.5秒内平滑降低到Miss对应的速度值，而非瞬间归零

### 需求:基础速率配置
系统必须支持为不同曲目配置不同的基础速率。基础速率决定了在全Perfect情况下完成比赛所需的时间。赛道长度固定为50米。

#### 场景:基础速率与曲目关联
- **当** 加载某首曲目的比赛配置
- **那么** 系统必须读取该曲目对应的基础速率参数，确保全Perfect完赛时间在曲目时长的85%-95%范围内

#### 场景:基础速率保障完赛
- **当** 玩家以平均Good的水平完成全曲
- **那么** 该玩家必须能在曲目结束前游完赛道的至少70%距离
