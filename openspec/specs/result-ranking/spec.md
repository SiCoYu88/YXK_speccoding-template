## 新增需求

### 需求:比赛结束判定
系统必须在以下任一条件满足时结束比赛：所有玩家到达终点、曲目播放完毕、比赛时间达到上限（曲目时长+30秒）。

#### 场景:全部到达终点
- **当** 所有存活玩家均到达50米终点
- **那么** 系统必须立即结束比赛并进入结算流程

#### 场景:曲目播放完毕
- **当** 音乐播放到末尾且仍有玩家未到终点
- **那么** 系统必须继续比赛，给予剩余玩家最多30秒额外时间到达终点

#### 场景:时间耗尽强制结束
- **当** 额外时间用尽仍有玩家未到终点
- **那么** 系统必须强制结束比赛，未到终点玩家以当前已游距离参与排名

### 需求:排名计算规则
系统必须按以下优先级进行排名：先按到达终点时间排序（先到先排）；未到终点的按已游距离排序；距离相同的按节奏准确率排序。

#### 场景:正常排名
- **当** 多位玩家在不同时间到达终点
- **那么** 系统必须按到达时间从早到晚排列名次（第1名到第N名）

#### 场景:未完赛玩家排名
- **当** 比赛结束时部分玩家未到终点
- **那么** 这些玩家必须排在所有完赛玩家之后，按已游距离从大到小排序

#### 场景:距离相同平局处理
- **当** 两名或以上未完赛玩家已游距离相同
- **那么** 系统必须以节奏准确率（Perfect+Great次数占总音符比例）高者排前

### 需求:结算界面展示
比赛结束后，系统必须展示结算界面，包含：排名列表、每位玩家的关键数据、个人详细统计。

#### 场景:排名列表展示
- **当** 结算界面加载
- **那么** 必须展示所有玩家的排名、名称、完赛时间（或已游距离）和节奏准确率

#### 场景:个人统计详情
- **当** 玩家查看个人统计
- **那么** 必须展示：总音符数、各判定等级数量及占比、最高连击数、冲刺段落得分、体力剩余值

#### 场景:MVP表彰
- **当** 结算完成
- **那么** 系统必须对第1名玩家播放特殊的MVP庆祝动画

### 需求:历史记录存储
系统必须保存玩家的比赛历史记录，包括每场比赛的排名、数据统计。最少保留最近50场记录。

#### 场景:比赛记录保存
- **当** 一场比赛结算完成
- **那么** 系统必须将本场比赛的完整统计数据存入玩家的历史记录

#### 场景:历史记录查询
- **当** 玩家在主界面查看历史记录
- **那么** 系统必须按时间倒序展示最近50场比赛的排名和关键数据摘要
