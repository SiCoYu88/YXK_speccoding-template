## 新增需求

### 需求:体力值管理
系统必须为每位玩家维护一个体力值，初始值为100，范围为0-100。体力值必须在比赛过程中随时间和操作变化。

#### 场景:比赛开始初始化体力
- **当** 比赛正式开始
- **那么** 所有玩家的体力值必须初始化为100

#### 场景:体力值边界约束
- **当** 体力计算结果超出0-100范围
- **那么** 系统必须将体力值钳制在0-100之间，禁止出现负值或超过上限

### 需求:体力消耗规则
系统必须按以下规则消耗体力：基础消耗为每秒消耗0.5点体力；Miss判定额外消耗3点体力；连续3次及以上Miss额外再消耗2点体力。

#### 场景:基础体力消耗
- **当** 比赛进行中每经过1秒
- **那么** 玩家体力必须减少0.5点（不受判定影响的固定消耗）

#### 场景:Miss额外消耗
- **当** 玩家获得Miss判定
- **那么** 在基础消耗之外，体力必须额外减少3点

#### 场景:连续Miss惩罚
- **当** 玩家连续获得3次或以上Miss判定
- **那么** 从第3次Miss开始，每次Miss必须额外再消耗2点体力（共额外消耗5点）

### 需求:体力对速度的影响
系统必须根据当前体力值计算体力系数。体力系数规则：体力>60时系数为1.0；体力30-60时系数为0.8；体力10-30时系数为0.6；体力<10时系数为0.4。

#### 场景:体力充沛无影响
- **当** 玩家体力值大于60
- **那么** 体力系数必须为1.0，不影响泳速计算

#### 场景:体力过低速度惩罚
- **当** 玩家体力值降至10以下
- **那么** 体力系数必须降为0.4，玩家泳速大幅下降

#### 场景:体力恢复后系数回升
- **当** 玩家体力从低于30恢复到30以上
- **那么** 体力系数必须相应回升，泳速在0.5秒内平滑恢复

### 需求:体力恢复机制
系统必须提供有限的体力恢复手段：连续10次Perfect判定恢复5点体力；连击达到50时一次性恢复10点体力。恢复量不得超过体力上限。

#### 场景:连续Perfect恢复
- **当** 玩家连续获得10次Perfect判定
- **那么** 系统必须恢复5点体力，并显示恢复特效提示

#### 场景:高连击奖励恢复
- **当** 玩家连击数达到50
- **那么** 系统必须一次性恢复10点体力，并显示里程碑恢复特效

### 需求:体力UI显示
系统必须在玩家HUD上显示当前体力值，以进度条形式展示。体力值低于30时必须以警告色（红色）闪烁提示。

#### 场景:正常体力显示
- **当** 比赛进行中且体力>30
- **那么** 体力条必须以绿色/蓝色显示当前体力百分比

#### 场景:低体力警告
- **当** 玩家体力降至30以下
- **那么** 体力条必须变为红色并以每秒2次的频率闪烁，同时播放警告音效
